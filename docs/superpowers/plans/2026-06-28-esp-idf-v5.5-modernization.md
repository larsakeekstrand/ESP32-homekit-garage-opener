# ESP-IDF v5.5.2 Modernization & Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the ESP32 HomeKit garage-door-opener firmware to ESP-IDF v5.5.2 and split the single `app_main.c` into focused modules, fixing known bugs along the way — with no new features.

**Architecture:** Re-clone `esp-homekit-sdk` as a git peer; pull the DHT driver via the v5.5 Component Manager (`idf_component.yml`); CMake-only build. `app_main.c` becomes a thin composition root that orchestrates four focused modules: `board.h` (pins), `hap_garage` (all HomeKit I/O), `door_control` (door state machine + relay + reed sensor), and `dht_sensor` (temp/humidity task). `hap_garage` and `door_control` are mutually referencing C translation units (HomeKit write → door request; door state change → HomeKit notify) wired through their headers.

**Tech Stack:** C, ESP-IDF v5.5.2, FreeRTOS, Espressif HomeKit SDK (`hap*`, `app_wifi`), esp-idf-lib DHT driver, CMake + ESP Component Manager.

## Testing Note (read before starting)

This firmware has **no automated test harness** and is hardware-bound (CLAUDE.md: "no tests, linters, or CI"). Per the approved spec, verification is:
- **Per task:** a clean `idf.py build` (zero errors) is the objective gate. Where a task changes observable behavior, the step says exactly what to confirm.
- **Final (Task 7):** on-device smoke test run by the user via `idf.py flash monitor`.

Do **not** invent unit tests for this codebase — there is no framework and the logic is I/O-bound on GPIO/HAP. The build is the test cycle.

**Build command used throughout** (sources the toolchain, then builds):
```bash
. ~/esp/esp-idf/export.sh && idf.py build
```
First build in a fresh checkout also needs `idf.py set-target esp32` (Task 1).

## Global Constraints

- **ESP-IDF version:** v5.5.2 exactly (installed at `~/esp/esp-idf-v5.5.2`, symlinked `~/esp/esp-idf`).
- **Target chip:** `esp32` (classic) only. Do not add S2/C3 target support.
- **Workspace layout (verified):** repo lives at `~/source/ESP32-homekit-garage-opener`, a **sibling** of `~/esp/` (NOT nested inside it, despite what the old README/CLAUDE.md say). `esp-idf` is at `~/esp/esp-idf` (→ `esp-idf-v5.5.2`); `esp-homekit-sdk` is re-cloned at `~/esp/esp-homekit-sdk`. `CMakeLists.txt` honors the `HOMEKIT_PATH` env override, else falls back to `${CMAKE_CURRENT_LIST_DIR}/../../esp/esp-homekit-sdk` (which resolves to `~/esp/esp-homekit-sdk` from this checkout).
- **Pin map (unchanged):** reset/button GPIO0, running LED GPIO2, door LED GPIO16, relay GPIO5, reed sensor GPIO4, DHT22/AM2301 GPIO22.
- **Relay/timing (unchanged):** 300 ms relay kick, 8 s (`DOOR_MOVING_MAXTIME` = 8000000 µs) failsafe timer.
- **No new features.** Only the two behavior changes the spec authorizes: reed-sensor debounce, and a working on-demand `dht_read`.
- **Commit after every task.** Branch is `modernize-esp-idf-v5.5` (already created).
- **Logging:** keep `ESP_LOGI`/`ESP_LOGE` with the `"HAP Garage"` tag style.

// door_control.c — door state machine, relay, reed sensor, failsafe timer.
//
#include <freertos/FreeRTOS.h>
#include <freertos/task.h>
#include <freertos/queue.h>
#include <driver/gpio.h>
#include <esp_log.h>
#include <esp_timer.h>

#include "door_control.h"
#include "hap_garage.h"
#include "board.h"

static const char *TAG = "HAP Garage";

static QueueHandle_t gpio_evt_queue = NULL;
static uint8_t door_state = HOMEKIT_CHARACTERISTIC_CURRENT_DOOR_STATE_UNKNOWN;
static esp_timer_handle_t oneshot_timer;

uint8_t door_get_state(void) { return door_state; }
static void door_set_state(uint8_t s) { door_state = s; }

/* Reed switch: hardware reads inverted, so XOR to logical DOOR_OPEN/DOOR_CLOSED. */
static uint8_t read_door_sensor(void) { return gpio_get_level(DOOR_SENSOR_GPIO) ^ 1; }

static void kick_door_relay(void) {
  gpio_set_level(DOOR_RELAY_GPIO, 1);
  vTaskDelay(300 / portTICK_PERIOD_MS);
  gpio_set_level(DOOR_RELAY_GPIO, 0);
}

static void timer_callback(void *args) {
  ESP_LOGI(TAG, "Timer expired. Setting state according to sensor.");
  if (read_door_sensor() == DOOR_OPEN) {
    hap_garage_send_target_door_state(HOMEKIT_CHARACTERISTIC_TARGET_DOOR_STATE_OPEN);
    hap_garage_send_current_door_state(HOMEKIT_CHARACTERISTIC_CURRENT_DOOR_STATE_OPEN);
    door_set_state(HOMEKIT_CHARACTERISTIC_CURRENT_DOOR_STATE_OPEN);
  } else {
    hap_garage_send_target_door_state(HOMEKIT_CHARACTERISTIC_TARGET_DOOR_STATE_CLOSED);
    hap_garage_send_current_door_state(HOMEKIT_CHARACTERISTIC_CURRENT_DOOR_STATE_CLOSED);
    door_set_state(HOMEKIT_CHARACTERISTIC_CURRENT_DOOR_STATE_CLOSED);
  }
}

static const esp_timer_create_args_t oneshot_timer_args = {
  .callback = &timer_callback, .name = "one-shot"
};

static void IRAM_ATTR gpio_isr_handler(void *arg) {
  uint32_t gpio_num = (uint32_t) arg;
  xQueueSendFromISR(gpio_evt_queue, &gpio_num, NULL);
}

- [ ] **Step 4: Add the consolidated, debounced event loop**

This replaces the original ~80-line, 8-branch loop. The reconciliation rule is: on a *debounced* sensor change, drive HomeKit to match the sensor, and clear any transient OPENING/CLOSING once the sensor settles. Behavior matches the original outcomes (manual or HomeKit-initiated both converge to the sensor truth) but in one readable block.

```c
/* Ignore reed edges that arrive within this window of the last accepted edge. */
#define DOOR_DEBOUNCE_US 50000

void door_control_run_loop(void) {
  uint32_t io_num;
  uint8_t last_sensor = read_door_sensor();
  int64_t last_change_us = 0;

  while (1) {
    if (xQueueReceive(gpio_evt_queue, &io_num, portMAX_DELAY) == pdFALSE) {
      ESP_LOGI(TAG, "Door trigger FAIL");
      continue;
    }

    int64_t now = esp_timer_get_time();
    if (now - last_change_us < DOOR_DEBOUNCE_US) {
      continue;  // debounce: too soon after the last accepted edge
    }

    uint8_t sensor = read_door_sensor();
    if (sensor == last_sensor) {
      continue;  // spurious edge, no real change
    }
    last_sensor = sensor;
    last_change_us = now;

    ESP_LOGI(TAG, "Door sensor %s (state was %s)",
             sensor == DOOR_OPEN ? "open" : "closed",
             door_get_state() == HOMEKIT_CHARACTERISTIC_CURRENT_DOOR_STATE_OPEN ? "open" : "not-open");

    if (sensor == DOOR_OPEN) {
      gpio_set_level(DOOR_LED_GPIO, 1);
      hap_garage_send_target_door_state(HOMEKIT_CHARACTERISTIC_TARGET_DOOR_STATE_OPEN);
      hap_garage_send_current_door_state(HOMEKIT_CHARACTERISTIC_CURRENT_DOOR_STATE_OPEN);
      door_set_state(HOMEKIT_CHARACTERISTIC_CURRENT_DOOR_STATE_OPEN);
    } else {
      gpio_set_level(DOOR_LED_GPIO, 0);
      hap_garage_send_target_door_state(HOMEKIT_CHARACTERISTIC_TARGET_DOOR_STATE_CLOSED);
      hap_garage_send_current_door_state(HOMEKIT_CHARACTERISTIC_CURRENT_DOOR_STATE_CLOSED);
      door_set_state(HOMEKIT_CHARACTERISTIC_CURRENT_DOOR_STATE_CLOSED);
    }
  }
}
```
